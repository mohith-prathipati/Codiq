# ⬡ Codiq Engine

**AI-powered verbatim coding platform for qualitative research**

Codiq automates the most time-consuming part of survey analysis — reading thousands of open-ended responses and assigning codes. It runs entirely on your machine using local AI, so your data never leaves your environment.

## What is Codiq?

Verbatim coding is a qualitative research method where open-ended survey responses are grouped into themes and assigned numeric codes — used in market research, UX research, and academic studies. Tools like Ascribe charge thousands of dollars and require sending your data to external servers.

Codiq does the same thing locally, free, with AI assistance.

---

## Features

### Two Coding Modes

| | Manual | Automated |
|---|---|---|
| **How it works** | You read responses and build the codebook on the fly | AI discovers themes, assigns codes, you review |
| **Codebook** | Created by you as you read | Auto-generated by Mistral |
| **Best for** | Small studies, full control | Large studies (500–2000+ responses) |
| **Output** | Coded Excel file | Coded Excel file |

### Core Capabilities

- **Unified workspace** — read responses and assign codes on the same screen, codebook always visible on the right
- **Hierarchical codebook** — themes grouped into NET categories, collapsible sections, usage counts shown per theme
- **Multi-code assignment** — up to 4 codes per response
- **Sentiment tagging** — Positive / Negative / Mixed per response
- **Reserved codes** — 9996 (Other), 9997 (Nothing), 9998 (Don't Know), 9999 (Can't Code) always present
- **Export to Excel** — 2-sheet output: Sheet1 = codebook, Sheet2 = coded responses with sentiment
- **Reusable codebooks** — save and reload codebooks across studies
- **Runs 100% locally** — no API keys, no data upload, no subscription

## Using Codiq

### Manual Coding

1. Select **Manual Coding** on the landing screen
2. Upload your Excel file (columns: RESPID, QID, Verbatim)
3. Click **Open Coding Workspace**
4. Read responses on the left
5. Click **+ Add Theme** on the right to create a theme as you discover it — select a NET category, type the theme name, press Enter
6. Click any theme to assign it to the selected response
7. Set sentiment (Positive / Negative / Mixed) for each response
8. Click **⬇ Export Excel** when done

### Automated Coding

1. Select **Automated Coding** on the landing screen
2. Upload your Excel file
3. Enter the survey topic (e.g. *"Coca-Cola Zero Sugar"*, *"Electric Vehicles"*)
4. Set the minimum responses threshold per theme
5. Click **▶ Run AI Coding** — the pipeline runs in the background
6. When complete, the workspace opens pre-filled with AI-assigned codes
7. Review, correct, add or remove codes
8. Click **⬇ Export Excel** when satisfied

## Input File Format

Your Excel file must have at least 3 columns:

| Column | Description | Example header names |
|---|---|---|
| Response ID | Unique identifier per respondent | `RESPID`, `resp_id`, `response_id` |
| Question ID | The survey question text or ID | `QID`, `question_id`, `q_id` |
| Verbatim | The open-ended response text | `Verbatim`, `Response`, `Text`, `Answer` |

Column names are auto-detected — exact naming doesn't matter as long as the intent is clear.

---

## Output File Format

The exported Excel has two sheets:

**Sheet 1 — Codebook**
- Survey question at the top
- Themes grouped by NET with code numbers
- Separated into Positives and Negatives sections
- Reserved codes at the bottom

**Sheet 2 — Coded Responses**
- All original columns preserved
- `Sentiments` column (Positive / Negative / Mixed)
- `Code_1` through `Code_4` columns

## AI Pipeline (Automated Mode)

The pipeline runs in 6 stages:

```
1. Discover    Sample responses in chunks → extract raw themes
2. Consolidate Merge near-duplicate themes → clean codebook
3. NET Codes   Assign each theme to a NET group + 3-digit code
4. Keywords    Generate keyword map for each theme (topic-aware)
5. Code        Assign codes to all responses in batches (2-pass)
6. Validate    Auto-correct known errors (ghost codes, bad sentiment, missing codes)
```

All stages run locally via Ollama. For 1000–1500 responses expect 30–60 minutes on Apple Silicon.

### NET Number Ranges

| Range | Category |
|---|---|
| 100–199 | Core Product Attributes (Positive) |
| 200–299 | Functional Benefits |
| 300–399 | Emotional Benefits |
| 400–499 | Brand & Trust |
| 500–599 | Design & Aesthetics |
| 600–699 | Value & Pricing |
| 700–799 | Naming & Communication Issues |
| 800–899 | Functional Concerns |
| 900–999 | Emotional Concerns |
| 1000–1099 | Brand & Trust Concerns |
| 1200–1299 | General Negative Sentiment |
| 1400–1499 | Target Audience Limitations |

### Reserved Codes

| Code | Meaning |
|---|---|
| 9996 | Other — has content but matches no theme |
| 9997 | Nothing / None / N/A |
| 9998 | Don't Know / Not Sure |
| 9999 | Can't Code — gibberish or unreadable |

## Configuration

All tunable parameters are at the top of `backend/pipeline.py`:

```python
CHUNK_SIZE        = 50    # responses per discovery chunk
MAX_SAMPLE_CHUNKS = 10    # chunks sampled for codebook build
CODE_BATCH        = 15    # responses per assignment batch
MAX_CODES         = 4     # max codes per response
MODEL             = "mistral"  # Ollama model to use
```

**To get more themes** (target 35–45):
```python
CHUNK_SIZE        = 30
MAX_SAMPLE_CHUNKS = 24
```

**To use a faster/different model:**
```python
MODEL = "mistral-nemo"   # faster on Apple Silicon
MODEL = "llama3.1"       # higher quality, slower
```

## API Reference

The FastAPI backend exposes these endpoints:

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/upload` | Upload Excel file, returns file metadata + preview |
| `POST` | `/run` | Start AI pipeline, returns `job_id` |
| `GET` | `/status/{job_id}` | Poll job progress, stage, log, codebook |
| `GET` | `/download/{job_id}` | Download coded Excel (pipeline output) |
| `POST` | `/export` | Export workspace state as coded Excel |
| `DELETE` | `/job/{job_id}` | Cancel a running job |

Interactive docs available at `http://localhost:8000/docs` when backend is running.

## Quality

The automated pipeline goes through multiple accuracy improvements:

- **Two-pass assignment** — responses initially coded as "Other" get a second attempt
- **Post-assignment validator** — auto-corrects ghost codes, invalid sentiments, missing obvious codes
- **Dynamic keyword injection** — topic-aware keyword map catches informal phrasings
- **Codebook deduplication** — merges overlapping themes post-assignment

Typical accuracy on consumer studies: **88–92%** vs human-coded ground truth.
